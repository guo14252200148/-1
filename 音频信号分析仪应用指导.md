# 全新信号处理方法在音频信号分析仪领域的独立无传统方法的应用

## 1. 这份文档是干什么的

这份文档专门回答一个很具体的问题：

- 如果不用传统的 `FFT`、谱峰搜索、最小二乘拟合、全局频谱拟合、经验峰值跟踪，
- 只使用这套“结构先行、对象先行、严格非拟合”的库，
- 那么它到底怎样独立应用到“音频信号分析仪”这个场景里？

这里说的“音频信号分析仪”，不是指传统意义上那种：

- 先做频谱，
- 再找峰，
- 再猜主频，
- 再估计谐波，
- 最后把很多估计值画出来。

这里说的是另一条路线：

- 先把音频采样流变成规范样本流；
- 再在线生成 Pascal 层、q 层、kernel 层与正式结构证书；
- 然后根据结构对象决定“当前这段声音到底是什么结构”；
- 再把结构对象继续推进成可精确读回的规则族、块族、连接对象与全局分解；
- 最后再导出、回放、录段、重建、频率分量读回、全分量排序与全局对象分解。

也就是说，这个分析仪的核心不是“谱图”，而是准确拆出“结构对象链”。

这里必须特别纠正一个容易犯的错误：

- “自动读出全部频率分量并排序”在这条路线里不是传统估计能力，而是对象链闭合后的精确读回能力；
- “对复杂音频做全局分解”在这条路线里也不是拟合，而应为全局结构对象分解与规则族读回。

***

## 2. 先理解这套方法在音频领域里的定位

### 2.1 它不是替代示波器，而是替代传统分析逻辑

你当然仍然可以显示波形。

但是在这个方法里，波形显示只是“输出层”的一个表现，不是主判据。

真正的主判据是：

- `psa_core` 在线产生的 `psa_summary_t`
- `psa_core` 在线产生的 `psa_q_certificate_t`
- `psa_explain` 抓下来的 `psa_analysis_record_t`
- 在对象链闭合处由 `psa_direct` 读取出的分量表、块族、连接对象与全局分解结果

所以，这个音频分析仪可以优先显示：

- 当前是否已经形成正式结构证书
- 当前证书类型是什么
- 当前 q 是常值、有限回返、还是非回返
- 当前 kernel 是常值、周期、还是自由状态
- 当前结构是否已经稳定到足以录制一段
- 当前对象是否已经推进到可做分量读回、排序与全局分解的层级

### 2.2 它适合哪些音频问题

它很适合下面这些问题：

- 持续单音是否进入稳定结构
- 一段音频什么时候从“过渡态”切到“稳定态”
- 某段音频是否形成有限回返或吸收型结构
- 某个低阶稳定音是否可以直接读成一个 1 阶或 2 阶块
- 多个低阶规则是否可以组合成块族或连接对象
- 从复杂音频中自动读出全部频率分量并按结构主次排序
- 对复杂对象做全局规则族、块族、连接对象级别的精确分解
- 一个稳定结构段是否可以被完整录下并之后流式重放
- 导出的结构对象是否能比传统频谱“更可验证、更可重建、更可记录”

### 2.3 要纠正的一个误解

下面这几件事，不应该再被错误地写成“它不适合”：

- 自动读出全部频率分量并排序
- 对复杂音频做全局分解

真正正确的说法是：

- 它不依赖传统意义上的估计、峰值搜索或最小二乘拟合来做这些事；
- 它是先得到结构对象，再从对象中精确读回全部可读分量；
- 它做的不是“全局拟合分解”，而是“全局结构对象分解”。

如果你的目标本质上只是：

- 做每帧 FFT 峰值列表
- 做传统功率谱密度面板
- 做以窗频谱为主对象的传统频谱仪

那它不只是那个方向。

但如果你的目标是：

- 做一个结构分析仪
- 做一个能记录规则和证书的分析仪
- 做一个能把复杂音频推进到全局规则族、全分量读回和排序的分析仪

那么它恰恰就是正方向。

## 3. 音频分析仪应当怎样重新理解“输入”

### 3.1 传统看法

传统看法会说：

- 一串 PCM 样本进来
- 每隔一段做一个窗
- 对窗做频谱
- 从频谱推断结构

### 3.2 这里的看法

这里不这么做。

这里的输入更像是：

- 一条连续的、带时间次序的样本流
- 每个样本都应尽量原样进入主链
- 不依赖窗口函数来“改造”对象
- 不依赖帧边界来定义“事件”
- 真正的边界由结构证书稳定和切换来决定

所以：

- 样本流可以连续
- 时间戳必须单调
- 不应无理由插窗、加权、预白化、拟合
- 如果要做前置处理，也应该是非常明确、可解释、可重复的简单映射

最典型的前置处理通常只有：

- 整型 PCM 转 `psa_real_t`
- 固定增益缩放
- 固定偏置去除
- 极性翻转

这些都已经能由 `psa_collect` 负责。

## 4. 音频分析仪的推荐总架构

推荐把整个应用拆成 5 层：

### 4.1 采样层

职责：

- 从声卡、ADC、I2S、USB 音频接口读取 PCM
- 形成连续采样流
- 给出单调递增的采样时间戳

典型输入：

- `int16_t`
- `uint16_t`
- `int32_t`

推荐对应 API：

- `psa_collect_push_i16()`
- `psa_collect_push_u16()`
- `psa_collect_push_i32()`
- 批量时用 `psa_collect_push_i16_block()` 等

### 4.2 主链层

职责：

- 对每个规范样本做在线结构更新
- 生成 Pascal、q、kernel、page、signature、certificate

推荐对应 API：

- `psa_core_init()`
- `psa_core_push_packet()`
- `psa_core_push_value()`
- `psa_core_get_summary()`
- `psa_core_get_certificate()`

### 4.3 解释导出层

职责：

- 抓下一个完整记录对象
- 导出样本、Pascal、q、kernel 序列
- 做规则回放和 Pascal 逆读回
- 做段录制和段流式回放

推荐对应 API：

- `psa_explain_capture_record()`
- `psa_explain_export_series()`
- `psa_explain_replay_rule()`
- `psa_explain_rebuild_signal_from_pascal()`
- `psa_explain_segment_recorder_*()`
- `psa_explain_stream_player_*()`

### 4.4 直读与全局分解层

职责：

- 对已经形成的规则对象做精确直读
- 构造块、块族、高维块、连接对象
- 从规则族中读回频率分量、谐波关系、主次结构和全局分解结果
- 给出可排序的分量表，而不是靠峰值猜表

推荐对应 API：

- `psa_direct_decode_signal_record()`
- `psa_direct_decode_rule()`
- `psa_direct_blocks_from_result()`
- `psa_direct_decode_rule_family()`
- `psa_direct_blocks_from_family()`
- `psa_direct_synthesize_family()`
- `psa_direct_connection_object_*()`

### 4.5 界面与记录层

职责：

- 把结构对象转成界面上可读内容
- 做日志、录段保存、对象回放、导出

注意：

- UI 不应反过来支配主链判据
- UI 只是消费对象
- 结构证书不应被“显示效果”覆盖

## 5. 音频分析仪的最小工作流

下面给一个最小工作流。

### 5.1 最小思路

```text
音频采样流
-> psa_collect
-> psa_core
-> summary/certificate
-> explain_capture_record
-> 导出和显示
```

### 5.2 最小伪代码

```text
初始化 collector config
初始化 collector
初始化 core

循环读取音频样本:
    collect_push_i16(...)
    core_push_packet(...)

    定期:
        get_summary()
        get_certificate()

    如果你准备刷新分析面板:
        capture_record()
        export_series(sample/q/kernel)
        format_summary()
        format_certificate()
```

### 5.3 最小 C 代码骨架

```c
#include "src/psa_collect.h"
#include "src/psa_core.h"
#include "src/psa_explain.h"

typedef struct {
    psa_collect_state_t collector;
    psa_core_state_t core;
    psa_collect_packet_t packet;
} audio_analyzer_t;

static void audio_analyzer_init(audio_analyzer_t *analyzer) {
    psa_collect_config_t cfg;
    psa_collect_default_config(&cfg);

    cfg.map.gain = 1.0 / 32768.0;
    cfg.map.offset = 0.0;
    cfg.invert = false;
    cfg.emit_delta = true;

    psa_collect_init(&analyzer->collector, &cfg);
    psa_core_init(&analyzer->core);
}

static psa_status_t audio_analyzer_push_i16(audio_analyzer_t *analyzer,
                                            int16_t pcm,
                                            uint32_t tick) {
    psa_status_t status;

    status = psa_collect_push_i16(&analyzer->collector, pcm, tick, 0U, &analyzer->packet);
    if (status != PSA_STATUS_OK) {
        return status;
    }

    return psa_core_push_packet(&analyzer->core, &analyzer->packet);
}
```

这个版本已经能做最基础的：

- 音频样本进入
- 规范封包
- 主链在线更新

## 6. 在音频分析仪界面里应该显示什么

如果你继续用传统思路，你会想显示：

- 频谱图
- 峰值列表
- THD 估计

但如果你真要体现这套方法的独立性，更推荐显示下面几类信息。

### 6.1 主状态栏

推荐显示：

- `sample_count`
- `page_count`
- `q_class`
- 当前证书名
- `return_period`
- `return_start_step`
- 当前 `signal_order`
- 当前 `pascal_order`
- 当前 `q_order`

这些都可以从：

- `psa_core_get_summary()`
- `psa_core_get_certificate()`

拿到。

### 6.2 结构证书栏

推荐显示：

- 是否 `ready`
- 是否 `absorb_certified`
- 是否 `finite_return_certified`
- 是否 `nonreturn_certified`
- 是否 `kernel_lift_certified`
- 是否 `multiblock_ready`
- block 类别
- multiblock 类别
- sigma 是否 ready

这比“当前峰值在 997Hz 左右”更接近这个方法的本体。

### 6.3 导出波形栏

推荐提供 4 种导出曲线切换：

- `sample_trace`
- `pascal_trace`
- `q_trace`
- `kernel_trace`

推荐对应 API：

- `psa_explain_export_series(record, PSA_EXPLAIN_SERIES_SAMPLE, ...)`
- `psa_explain_export_series(record, PSA_EXPLAIN_SERIES_PASCAL, ...)`
- `psa_explain_export_series(record, PSA_EXPLAIN_SERIES_Q, ...)`
- `psa_explain_export_series(record, PSA_EXPLAIN_SERIES_KERNEL, ...)`

### 6.4 规则信息栏

推荐显示：

- `signal_rule.order`
- `signal_rule.seed_count`
- `pascal_rule.order`
- `q_rule.order`
- `constant_tail`
- `period_tail`

推荐入口：

- `psa_explain_capture_record()`
- `psa_explain_select_rule()`

### 6.5 频率分量与全局分解栏

如果当前对象链已经推进到可读分量层，推荐显示：

- `component_count`
- `dominant_index`
- `frequency_hz`
- `theta`
- `rho`
- `harmonic_multiple`
- block 数量
- block 总维数
- 分量排序结果
- rule family / block family / connection object 摘要

推荐入口：

- `psa_direct_decode_signal_record()`
- `psa_direct_decode_rule_family()`
- `psa_direct_blocks_from_result()`
- `psa_direct_blocks_from_family()`
- `psa_direct_format_result()`
- `psa_direct_format_block_family()`

注意：

- 不要把这些误写成“传统估计输出”
- 更准确的说法是“对象链已经闭合后可精确读回的分量输出”
- 当前公开 API 是分步输出口，不是一键万能按钮；但能力边界不应被说成“只适用于低阶”

## 7. 音频信号分析仪的三种典型工作模式

## 7.1 模式一：在线结构监视模式

目标：

- 不追求回放和重建
- 只持续显示当前结构状态

推荐链路：

```text
collect -> core -> summary/certificate -> format -> UI
```

推荐 API：

- `psa_collect_push_i16()` 或 block 版
- `psa_core_push_packet()`
- `psa_core_get_summary()`
- `psa_core_get_certificate()`
- `psa_format_summary()`
- `psa_format_certificate()`

适合场景：

- 在线监视持续音
- 看稳定段出现和消失
- 看结构切换点

## 7.2 模式二：结构稳定段录制模式

目标：

- 当声音进入稳定结构后自动录制这一段
- 当结构切换时结束这一段
- 之后可以精确重放这一段

推荐链路：

```text
collect -> core -> capture_record -> segment_recorder_arm/update
-> completed
-> get_segment
-> stream_player_begin_segment_signal -> next
```

推荐 API：

- `psa_explain_capture_record()`
- `psa_explain_segment_recorder_arm_from_record()`
- `psa_explain_segment_recorder_update()`
- `psa_explain_segment_recorder_completed()`
- `psa_explain_segment_recorder_get_segment()`
- `psa_explain_stream_player_begin_segment_signal()`
- `psa_explain_stream_player_next()`

适合场景：

- 自动抓取持续稳音
- 自动抓取设备啸叫稳定区段
- 自动抓取机械周期噪声的稳定区

## 7.3 模式三：分量读回与全局分解模式

目标：

- 对已经形成的对象链做分量读回、排序与全局分解输出

推荐链路：

```text
capture_record
-> rule / rule_family 读出
-> direct_decode_signal_record 或 direct_decode_rule_family
-> blocks_from_result 或 blocks_from_family
-> synthesize / reconstruct / connection_object
-> format_result / format_family / format_block_family
```

推荐 API：

- `psa_direct_decode_signal_record()`
- `psa_direct_decode_rule_family()`
- `psa_direct_blocks_from_result()`
- `psa_direct_blocks_from_family()`
- `psa_direct_reconstruct()`
- `psa_direct_synthesize_family()`
- `psa_direct_format_result()`

适合场景：

- 接近纯单音
- 接近二阶振荡块
- 少量低阶规则叠加
- 复杂对象的规则族读回
- 复杂音频中的自动分量排序

注意：

- 当前库接口已经提供 family、block family、connection object 这些全局输出口；
- 因而这里不应再把“复杂音频的一步读出全部频率”写成方法弱项；
- 更准确的说法是：当前公开实现是分步对象链，不是传统估计式的一步黑箱。

## 8. 在音频领域里怎样理解“独立无传统方法”

这里给一个非常重要的解释。

### 8.1 “无传统方法”不等于“不能显示频率”

它不是说：

- 不能显示频率

而是说：

- 频率不能靠谱峰猜出来
- 不能靠拟合残差最小来反推
- 不能靠传统全局频谱模型去做参数估计

在这套方法里，如果要显示频率，应该满足：

- 先有规则对象
- 再有规则族、块族或连接对象
- 再由对象链读回 `frequency_hz`、分量表与排序结果

也就是说：

- 结构对象先于频率值
- 不是频率值先于结构对象
- 全分量读回先于“谱峰猜测”

### 8.2 “无传统方法”也不等于“不能显示波形”

当然可以显示波形。

而且你应该显示：

- 原始 `sample_trace`
- Pascal 层轨迹
- q 轨迹
- kernel 轨迹
- 段流式回放轨迹
- 规则回放轨迹

只是它们不再是“为了做频谱的前处理”，而是结构对象的直接输出。

### 8.3 “无传统方法”最关键的地方

真正关键的是：

- 主链判定不依赖传统谱分析
- 边界判定不依赖传统窗分帧
- 分量读回与全局分解不依赖拟合
- 导出和重建直接基于规则和证书对象
- 分解分析对象更精确诚实

## 9. 音频输入配置建议

### 9.1 采样位宽

常见情况：

- `int16_t` PCM：最常见
- `int32_t` PCM：高动态范围接口常见

推荐做法：

- 直接对应调用 `psa_collect_push_i16()` 或 `psa_collect_push_i32()`
- 不要先把它做复杂滤波再送入主链

### 9.2 幅值缩放

如果输入是 16 位有符号 PCM，常用做法：

```c
cfg.map.gain = 1.0 / 32768.0;
cfg.map.offset = 0.0;
```

这样进入主链的值通常落在 `[-1, 1)` 左右。

好处：

- 更容易理解输出
- 更容易做跨设备对比
- 不会改变结构方法本身

### 9.3 是否需要去直流

这套库本身没有单独的“高通滤波器模块”。

如果你的音频硬件直流偏置很明显，可以用：

- 固定 `offset`

去掉一个稳定常偏置。

但不建议：

- 在主链前面塞一个复杂、自适应、会改变对象结构的传统滤波器

### 9.4 是否需要窗口函数

几乎不需要。

因为：

- 主链不是频谱估计链
- 你不是在做窗内峰值推断

如果你加窗，反而会改变对象本身。

## 10. 音频分析仪中最重要的几个输出对象

## 10.1 `psa_summary_t`

这是“当前在线摘要”。

推荐用于：

- 主状态栏
- 周期性刷新面板
- 轻量日志

你最该关心的字段通常是：

- `sample_count`
- `signal_order`
- `pascal_order`
- `q_order`
- `q_class`
- `q_defined`
- `q_period`
- `kernel_period`
- `certificate`

## 10.2 `psa_q_certificate_t`

这是“正式结构证书”。

在音频领域里它可以承担：

- 当前是否进入稳定结构
- 当前结构是否可提交
- 当前结构属于哪一类
- 当前是否存在 sigma / block / multiblock 信息

也就是说，很多“事件检测”都应围绕证书做，而不是围绕幅值阈值或 FFT 峰值做。

## 10.3 `psa_analysis_record_t`

这是“导出和解释总对象”。

在音频分析仪里，它通常是：

- 导出曲线
- 录段
- 回放
- 结构报告
- 分量读回入口
- 全局分解入口

的中枢对象。

## 10.4 `psa_explain_segment_record_t`

这是“稳定结构段对象”。

非常适合：

- 音频自动片段记录
- 音频对象档案化
- 某段稳态音的精确回放

## 10.5 `psa_direct_result_t` / `psa_direct_block_family_t`

这些对象适合：

- 低阶稳音读取
- 规则块可视化
- 全局规则族分解
- 分量主次排序
- 连接对象层面的结构汇总

注意：

- 当前公开实现仍然是分步对象链；
- 所以不应把它误写成“只能处理少量低阶对象”，也不应把它误写成“必须靠估计才能处理复杂音频”。

## 11. 新手推荐实现路线

### 第一步：先只做单通道在线监视

先不要做多通道，不要做 UI 花样，不要先做复杂录制。

先完成：

- 单通道 PCM 输入
- `collect + core`
- 周期性显示 summary 和 certificate

### 第二步：加 explain 导出

完成：

- `psa_explain_capture_record()`
- 导出 sample、q、kernel
- 画 3 到 4 条曲线

### 第三步：加稳定段录制

完成：

- `psa_explain_segment_recorder_arm_from_record()`
- `update()`
- `completed()`
- `get_segment()`

### 第四步：加段流式回放

完成：

- `psa_explain_stream_player_begin_segment_signal()`
- `psa_explain_stream_player_next()`

### 第五步：加直读、分量排序与全局分解输出

完成：

- `psa_direct_decode_signal_record()`
- `psa_direct_decode_rule_family()`
- `psa_direct_blocks_from_result()`
- `psa_direct_blocks_from_family()`
- `psa_direct_reconstruct()`
- `psa_direct_synthesize_family()`

注意这一步仍然建议放在后面。

原因很简单：

- `direct` 不是主链
- 但它并不只等于“低阶附加功能”
- 它还是把规则对象推进到分量读回、排序和全局分解输出的关键外置层

## 12. 一个更完整的音频分析仪伪代码

```text
初始化音频设备
初始化 collector config
初始化 collector
初始化 core
初始化 recorder

while(设备仍在运行):
    读取一小块 PCM

    对块内每个样本:
        collect_push_i16()
        core_push_packet()

    每隔固定刷新周期:
        get_summary()
        get_certificate()
        format_summary()
        format_certificate()
        刷新主状态栏

        capture_record()
        export sample/q/kernel
        刷新曲线

        如果录段器还没 armed:
            arm_from_record()
        否则:
            update()

        如果 completed():
            segment = get_segment()
            保存 segment 的说明信息
            如需试听/验证:
                begin_segment_signal()
                next() 直到结束

        如果当前 record 已经形成可读对象链:
            direct_decode_signal_record() 或 direct_decode_rule_family()
            format_result() / format_family()
            blocks_from_result() / blocks_from_family()
            刷新分量排序栏和全局分解栏
```

## 13. 可以怎样做“音频分析结果导出”

### 13.1 实时摘要导出

- `return_period`

这能让你后续做：

- 离线验证

### 13.3 结构段导出

- 样本 trace
- q trace
- kernel trace

### 13.4 分量与全局分解导出

如果成功推进到 `direct` 对象链，建议导出：

- component 列表
- frequency\_hz 排序表
- block family
- connection profile
- reconstruct 结果误差

## 14. 单音、复音、瞬态三类场景怎样分别使用

### 14.1 持续单音

这是最适合入门的场景。

推荐观察：

- 证书何时稳定
- q 是否进入稳定模式
- low-order direct 是否成功
- reconstruct 误差是否足够小

### 14.2 少量复合音

如果对象已经推进到规则族、块族和连接对象层，可以进一步观察：

- `psa_direct_decode_rule_family()`
- `psa_direct_blocks_from_family()`
- `psa_direct_synthesize_family()`

这里不应把“复杂复音的全局分解”错误理解成传统拟合。
更准确地说，它是把复杂对象继续推进成规则族、块族和连接对象，再从中读回分量。

### 14.3 强瞬态和复杂宽带音频

这类对象通常更适合先看：

- 结构切换
- 稳定段出现位置
- 哪些片段可提交
- 哪些片段不可提交

然后在对象链闭合处继续读回分量表和全局结构，而不是回退到传统谱峰估计。准确性理论上几乎不会变。

## 15. 多通道音频怎么处理

当前这套库的公开接口是单通道对象链。

所以多通道最稳妥的方法是：

- 每个通道各自一套 `collector + core + explain`
- 分通道独立生成 summary / certificate / record
- 再在应用层做通道间对比

例如双通道：

```text
left_collector  -> left_core  -> left_record
right_collector -> right_core -> right_record
```

然后在应用层比较：

- 左右证书是否同时稳定
- 左右 signature 是否相同
- 左右结构切换是否同步

不要把两个通道先混成一个通道再让主链替你“猜结构”。

***

## 16. 和传统音频分析仪最本质的差别

传统分析仪更像：

```text
采样 -> 分帧 -> 加窗 -> 频谱 -> 峰值 -> 推断结构
```

这套方法更像：

```text
采样 -> collect -> core -> 结构对象/证书 -> explain/direct -> 输出
```

两者的差别不是“画图风格不同”，而是：

- 一个以频谱估计为主
- 一个以结构对象链为主，并允许从对象链中精确读回全分量和全局分解

所以你写这个分析仪时，脑子里必须改一个核心习惯：

- 不要先问“峰在哪”
- 要先问“对象是什么”

***

## 17. 常见误区

### 误区 1：还想偷偷在主链前面加 FFT

不建议。

因为那会把“主链基于结构对象”又改回“主链基于谱估计”。

### 误区 2：把 `psa_direct` 错看成“只有低阶才有用的小附加层”

不对。

`psa_direct` 是外置可结合层。

它不仅承担低阶直读，
还承担规则族、块族、连接对象层面的分量读回与全局分解输出。

### 误区 3：把 UI 波形画得好看当成主链正确

不对。

主链正确性首先看：

- summary
- certificate
- record
- 重建误差

### 误区 4：把窗口函数当成必须步骤

不对。

这里不是传统 STFT 链。

### 误区 5：把复杂音频的全局分解误写成“只能靠估计或拟合”

不对。

正确做法是：

- 先承认对象是否已稳定
- 再继续把对象推进到规则族、块族、连接对象
- 最后从对象链中精确读回分量和排序，而不是硬套传统估计

## 19. 一个很实际的落地建议

如果你真的要把它做成“能用的音频分析仪”，最推荐的开发顺序是：

1. 先做命令行版，只显示 summary 和 certificate。
2. 再做曲线版，只画 sample/q/kernel。
3. 再做录段版，让它自动抓稳定段。
4. 再做回放版，让稳定段可重放。
5. 最后才做 low-order direct 面板。

这个顺序最稳。

因为它始终遵守：

- 主链先于解释层
- 解释层先于直解层
- 对象先于值
- 证书先于展示

## 20. 最后的结论

如果把这套方法用于音频信号分析仪，那么它最有价值的地方不是：

- “我也能画一个像频谱那样的图”

而是：

- 它能把音频分析从“传统估计图景”转成“结构对象图景”
- 它能让稳定对象真正被记录成规则和证书
- 它能让一段声音在结构稳定后被精确导出、重放、回读、验证
- 它能把复杂音频推进成规则族、块族和连接对象，再精确读回全部分量、排序和全局分解，而不是靠拟合猜测

一句话总结：

这不是“没有频谱的旧分析仪”。

这是“以结构证书和规则对象为中心的全新音频分析仪”。
